# 案由与分类索引

> **用途**：新建案件时，`case_type` 建议与**人民法院常用表述**或《民事案件案由规定》中的名称对齐，便于检索、立案与团队统一口径。  
> **不是封闭列表**：下列未涵盖的，直接写**实务中真实案由**即可；只有暂时无法归类时，才用 `其他` 或 `综合` 作大类标签（若你在模板中使用了 `case_domain`）。

---

## 一、`case_type` 怎么填

| 做法      | 说明                                                    |
| ------- | ----------------------------------------------------- |
| **建案时** | 由 **Agent 根据律师/当事人提供的事实自动推断**并写入 YAML；律师只需事后看一眼，有错再改。 |
| **优先**  | 写**具体案由**，如「民间借贷纠纷」「名誉权纠纷」「劳动争议」「执行异议之诉」等。            |
| **不必**  | 为「不在下面示例里」而强行选「其他」——「其他」适合**暂无法细分**或**多法律关系竞合**时暂用。   |
| **可改**  | 接洽过程中案由细化或法院立案案由调整后，直接改 YAML 与正文即可。                   |
| **有疑问** | Agent 在案件笔记中插入「案由待核对」提示即可，**不**把推断责任推回给律师当场填表。        |

---

## 二、大类与常见示例（可扩展）

以下为**便于统计或 Dataview 筛选**的域划分示例，与 `case_type` 具体字符串配合使用；你若未使用 `case_domain` 字段，可忽略本表第二列。

| case_domain（可选） | case_type 示例 |
|---------------------|----------------|
| 婚姻家事 | 离婚纠纷、离婚后财产纠纷、抚养纠纷、探望权纠纷、赡养纠纷、婚约财产纠纷 |
| 合同 | 买卖合同纠纷、民间借贷纠纷、租赁合同纠纷、服务合同纠纷、建设工程合同纠纷 |
| 侵权 | 机动车交通事故责任纠纷、医疗损害责任纠纷、名誉权纠纷、隐私权纠纷 |
| 物权 | 所有权纠纷、用益物权纠纷、担保物权纠纷、相邻关系纠纷 |
| 公司 | 股东资格确认纠纷、损害公司利益责任纠纷、公司决议纠纷 |
| 劳动 | 劳动争议、工伤保险待遇纠纷 |
| 房地产 | 房屋买卖合同纠纷、物业服务合同纠纷 |
| 知识产权 | 著作权侵权、商标权侵权、不正当竞争纠纷 |
| 金融 | 金融借款合同纠纷、信用卡纠纷 |
| 执行 | 执行异议之诉、追加被执行人 |
| 行政 | 行政处罚、行政复议（视业务是否代理） |
| 刑事 | （若本所代理）具体罪名由卷宗与起诉书表述为准 |
| 综合 / 其他 | 法律关系复杂、多案由并存或暂未拆分时使用 |

## 三、与案件笔记的链接

新建或更新案件时，可在案件正文「案件概要」中写明**法院案号立案案由**（如有），与 YAML 中 `case_type` 保持一致或互为补充。
